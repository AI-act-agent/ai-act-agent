# AI기본법 QA 에이전트 실험 기록

## 목적

V0부터 V4까지 에이전트가 개선되는 과정을 재현하고, 버전별·모델별 성능 차이를 발표 자료로 시각화하기 위한 기록이다.

## 기록 원칙

각 실험마다 다음 항목을 기록한다.

- 실험 ID와 날짜
- 코드 버전 또는 Git 커밋
- 사용 데이터
- 임베딩 모델
- 생성 모델
- 검색 및 에이전트 설정
- 테스트 질문
- 실제 결과
- 성공한 점과 실패한 점
- 다음 개선 사항

## 주요 평가 지표

- 검색 적중률: 정답 조문이 Top-K에 포함되는 비율
- 답변 정확도: 기대 답변과 일치하는 비율
- 인용 정확도: 답변과 인용 조문이 일치하는 비율
- Grounding 통과 정확도
- 판단 보류 정확도
- 평균 응답 시간
- 질문당 외부 API 호출 수

## 버전 정의

| 버전 | 핵심 기능 |
| --- | --- |
| V0 | 검색 없이 생성 모델만 사용한 기준선 |
| V1 | 임베딩 기반 기본 RAG 검색 |
| V2 | RAG 검색과 조문 참조 관계 확장 |
| V3 | 조사 계획 생성, 재검색, 판단 보류 |

- 대상 버전: V3

- 데이터: 가상 조문

- 측정 상태: 정량 지표 미측정

### EXP-003 — NLI 기반 Grounding 검사

- 대상 버전: V4
- NLI 모델: `MoritzLaurer/mDeBERTa-v3-base-mnli-xnli`
- 실행 방식: 로컬 CPU
- 추가 기능:
  - 생성 답변과 검색 근거의 논리적 일치 여부 검사
  - `entailment`만 답변 확정
  - `neutral` 또는 `contradiction`이면 재검색
- 확인 결과:
  - 근거와 일치하는 테스트 답변은 `entailment`로 판정됐다.
  - 근거와 반대되는 답변은 `contradiction`으로 판정됐다.
  - V4 정상 질문에서 Grounding 통과 후 답변이 확정됐다.
- 장점:
  - Grounding 검사에는 외부 API 할당량이 필요하지 않다.
  - 근거와 일치하지 않는 답변의 확정을 방지할 수 있다.
- 발견된 문제:
  - `python -c` 실행마다 NLI 모델을 다시 메모리에 불러와 초기 실행 시간이 길다.
  - 실제로 근거가 있는 답변도 `neutral`로 판단하는 거짓 거절이 발생할 수 있다.
- 측정 상태: 정량 지표 미측정
- 다음 개선: 실제 RAG 데이터 연결

## 2026-07-23 고영향 AI 사전 검토 Rule Baseline v0

### 목표

자연어로 설명된 AI 시스템의 운영 사실을 구조화하고, 고영향 AI 해당 가능성을 `해당 가능성 높음`, `해당 가능성 낮음`, `판단 보류`로 사전 검토하는 기능의 초기 기준선을 측정한다.

### 구현 내용

- 사전 검토 입력 및 결과 데이터 구조 정의
- 자연어 입력에서 운영 사실을 추출하는 Gemini 기반 Extractor 구현
- 부족한 정보를 한 번에 하나씩 묻는 멀티턴 재질문 루프 구현

### 오류 분석

- E001의 `ai_role`에서 `탈락`까지 요구했으나, 탈락은 `decision_consequence`와 `automatic_decision`에 이미 분리되어 있었다.
- E004의 `human_review_process`에서 `최종`을 요구했으나, 최종 결정 여부는 `human_final_decision`에 별도로 저장되어 있었다.
- E004의 `decision_consequence`가 null로 추출되어, AI 점수가 사람의 최종 채용 판단에 사용된다는 사실을 놓쳤다.

앞의 두 항목은 평가 기준이 필드 간 정보를 중복 요구한 문제였고, 마지막 항목은 실제 Extractor 프롬프트 개선 대상으로 분류하였다.

### Prompt v1 개선

- `ai_role`은 AI가 직접 수행하는 분석·요약·점수 산정·추천 작업으로 한정
- `decision_consequence`는 AI 결과가 평가나 최종 결정에 사용되는 방식을 기록
- `human_review_process`는 사람의 실제 검토 절차만 기록
- 최종 결정 및 결과 변경 가능 여부는 각각 Boolean 필드로 분리
- 명시적 부정은 false, 언급되지 않은 정보는 null로 구분
